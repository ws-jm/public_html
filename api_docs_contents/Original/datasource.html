3.  Datasource Metadata

Datasources

Our datasources are time series databases containing end of day market data from exchanges, commodity markets, stock markets, banks, governments, NGO’s, data vendors and independent pricing agencies from around the world. A datasource contains both datasets (historical time series) and metadata (information describing the datasource and its datasets).

The datasource endpoints allow you to get:

  • The metadata for one named datasource using the GetDatasource endpoint.
  • A metadata list of all the available datasources on the server using the GetAllDatasources endpoint.
  • A metadata list of accessible datasources (based on the subscription) using the GetUserDatasources endpoint.

Some datasources are subdivided and grouped into data categories for logical or commercial reasons. We refer to these types of datasource as “CategoryDS” datasources. Typically a user may not have access to all the data categories in a CategoryDS datasource. The data category name will need to be included in all dataset/value requests as it is possible that a symbol exists in more than one data category. CategoryDS datasources can optionally also return a category list (and tree) of all available data categories as well as the categories that the user can access in the datasource. See the “CategoryDS datasources” section below for more details.

To make it easier for users to locate the datasets they require, some datasource endpoints may return a hierarchical data tree that can be used to organize some or all of the the datasets by market, region, instrument type, exchange etc. If present, this tree can be used in 3rd party applications to help organise a non CategoryDS datasource.

Notes
  • A subscriber may not have access to the full data range available in any datasource, data category or dataset (subscription limitations).
  • The GetUserDatasources endpoint returns the date range that is available to the subscriber for each datasource and for the data categories in a CategoryDS datasource.
  • The GetDatasource endpoint returns the the metadata about a single datasource including any subscription access details.
  • Datasources can be defined as ‘premium’ or ‘complimentary’. Complimentary datasources may be freely available with a premium datasource.
  • The datasource metadata reports if the datasource is a “CategoryDS” datasource, has an optional data tree, is premium and the number of datasets/data categories in the datasource.
  • Use the Datasets metadata endpoints to view the list of datasets that are in any datasource.
CategoryDS datasources

  • A CategoryDS is a datasource whose datasets are grouped in data categories. The user must be authorised to access individual data categories.
  • Category datasources are defined in their metatata as “ISCategoryDS“: true and have a “CategoryCount” parameter value greater than zero.
  • The category name parameter must be used in all data requests to a CategoryDS datasource.
  • In GET requests Datasource, Datacategory (in a CategoryDS datasource only) and Symbol are combined into one ‘Series’ string.
    e.g. Symbols[]=Datasource/Datacategory/Symbol&Symbols[] = …
