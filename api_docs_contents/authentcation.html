2. Token Authentication

Token Authentication

The Monitor+ API uses ‘session’ tokens to authenticate and identify each user to the server. The user requests a session token using their private API key and, once a token has been received from the server, it can be used in all further endpoints (API requests). Tokens automatically expire after 20 minutes of inactivity but live tokens renew automatically to full life each time an endpoint that uses the token is used.

With the authentication endpoints you can create, revoke, extend or query the remaining life of a session token.

  • Use the GetSessionToken endpoint to get a new token (using an API key).
  • Use the RenewSessionToken endpoint to renew an existing token to full life.
  • Use the RevokeSessionToken endpoint to revoke or cancel an token.
  • Use the SessionTokenExpires endpoint to query the remaining life of an existing token.

Once you have authenticated and received a session token, use your session token with the metadata endpoints to locate all available datasources or only the datasources that your subscription entitles you to access.

Session tokens are required to get both price data and ‘favorites’ metadata from the server. Please refer to the documentation for each individual endpoint for more details.
